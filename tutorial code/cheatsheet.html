Data types and operations

Numbers

integer (int) and float, while ints do not have decimal places but floats do.

Operations:
Let x, y be an int or float.

  • x + x
  • x - x
  • x * x
  • x % y (modulus, returns the remainder of x / y)
  • x / x (returns a float)
  • x // x (returns an int with decimal places cut off)
  • x ** y (returns x to the power of y)

  • x < y (less than)
  • x <= y (less than or equal to)
  • x > y (greater than)
  • x >= y (greater than or equal to)
  • x == y (True if the values are equal)
  • x != y (True if the values are unequal, opposite to == )
  • x (evaluated to True if x is not None and x != 0) -> if while for
  • x is y (true if the object in memory both x and y reference is the same)

  • parentheses (), to mark or change the operator order
  • bitwise operations (~, |, &, <<, >>)

Boolean

A boolean (bool) can only have one of the two values: True or False.

logical operations:
Let b, i be bool variables.

  • b == i (True if both are either True or False)
  • b != i (True if the values are not the same)

  • b is i
  • b (True if b is True) -> if b: .. is possible with that instead of if b == True: ..
  • not b (True if b is False)

None

None is a type to signal that there is nothing, not even a zero (which would be an int or float, so a fixed value).

Operations:
Let x be any variable.

  • x == None
  • x != None
  • x is None

String

A string (str) is a series of characters (utf-8 encoded per default in python3). Starts and ends with a " or ', for multiple lines also with """

Operations:
Let x be an int.

  • "Hello" + " world" (returns the concatinated string "Hello world")
  • "a" * x (returns x-times the content of the string to a new one)

  • A nonempty string will always be evaluated to True: if "str": .. is the same as if "str" != "": ..

list comprehensions

Imagine you have got code like the following:

baseList = ["hello", 42, False, None, "you"]
itemList = []
for e in itemList:
  itemList.append(e)

The following does exactly the same:

baseList = ["hello", 42, False, None, "you"]
itemList = [e for e in baseList]

You may even use if-statements in that sweet little thing. Instead of writing:

baseList = ["hello", 42, False, None, "you"]
itemList = []
for e in itemList:
  if type(e) == str:
    itemList.append(e)

we can abbreviate to:

baseList = ["hello", 42, False, None, "you"]
itemList = [e for e in baseList if type(e) == str]

How cool & neat is that!?
